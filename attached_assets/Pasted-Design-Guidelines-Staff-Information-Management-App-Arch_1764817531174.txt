Design Guidelines: Staff Information & Management App

Architecture Decisions

Authentication

Auth Required - Individual staff authentication with PIN/codes

No SSO: Use custom PIN/code entry system (4-6 digit codes)
Login flow:
Initial screen: "Staff Login" button overlaying public content
PIN entry modal: numeric keypad with masked input (●●●●)
"Forgot Code?" link → contact management message
Auto-dismiss on successful auth, error shake animation on failure
Roles: Public, Staff, Management
Account persistence: Store encrypted credentials locally
Logout: Accessible from profile tab, with confirmation alert
Management can reset staff codes from their dashboard
Navigation

Hybrid: Tab Navigation + Drawer

Public Mode (no auth): Stack-only navigation through information screens
Staff Mode: 4-tab navigation
Home (information hub)
Messages (staff messaging)
Content (edit/manage content)
Profile (personal settings, logout)
Management Mode: 5-tab navigation (adds Management tab between Content and Profile)
Management tab opens drawer with:
Staff Directory
Add Staff Member
Permissions Manager
Role Assignments
Audit Log
Screen Specifications

Public Area

Information Hub Screen

Purpose: Browse public company information
Header: Custom, transparent, title "Information", right button "Staff Login"
Layout: ScrollView root
Components: Category cards, search bar, featured content carousel
Safe area: top = headerHeight + Spacing.xl, bottom = insets.bottom + Spacing.xl
Staff Area

Staff Home Screen

Purpose: Dashboard for staff with role-appropriate content
Header: Default, title "Dashboard", right button (notifications bell icon)
Layout: ScrollView with pull-to-refresh
Components: Quick action cards, recent updates list, restricted data widgets
Safe area: top = Spacing.xl, bottom = tabBarHeight + Spacing.xl
Messages Screen

Purpose: Staff-to-staff internal messaging
Header: Default, title "Staff Messages", right button "New Message" (+)
Layout: FlatList of conversation threads
Components: Message preview cards with avatars, unread badges, timestamps
Safe area: top = Spacing.xl, bottom = tabBarHeight + Spacing.xl
Content Management Screen

Purpose: Edit and publish information content
Header: Default, title "Manage Content"
Layout: Segmented control switching between "Published" and "Drafts" lists
Components: Content cards with edit/delete actions, floating action button "Create"
Safe area: bottom = tabBarHeight + Spacing.xl (FAB at bottom + 80)
Profile Screen

Purpose: Personal settings and account management
Header: Custom, transparent, title "Profile"
Layout: ScrollView
Components: Avatar, name, role badge, settings list, logout button (destructive)
Safe area: top = headerHeight + Spacing.xl, bottom = tabBarHeight + Spacing.xl
Management Area

Staff Directory Screen (Drawer item)

Purpose: View and manage all staff members
Header: Default, title "Staff Directory", right button "Add Staff" (+)
Layout: SearchBar + FlatList
Components: Staff cards showing name, role, status, last active
Safe area: top = Spacing.xl, bottom = tabBarHeight + Spacing.xl
Edit Staff Modal (opens from directory)

Purpose: Modify staff details, code, permissions, or remove
Layout: Scrollable form
Components:
Name input
Role picker (Staff/Management)
"Reset Code" button
Permissions toggles (View Restricted Data, Edit Content, Send Messages, Manage Staff)
"Remove Staff" button (destructive, bottom, double confirmation)
Buttons: Cancel (header left), Save (header right)
Add Staff Modal

Purpose: Create new staff account
Layout: Scrollable form
Components: Name input, role picker, generate code button, permissions toggles
Buttons: Cancel (header left), Create (header right)
Design System

Color Palette

Primary: #2563EB (professional blue for primary actions)
Secondary: #7C3AED (purple accent for management features)
Success: #10B981 (confirmations, active status)
Destructive: #EF4444 (delete, remove actions)
Background: #F9FAFB (light gray)
Surface: #FFFFFF (cards, modals)
Text Primary: #111827
Text Secondary: #6B7280
Border: #E5E7EB
Typography

Header Large: 34pt, Bold (screen titles)
Header: 20pt, Semibold (section headers)
Body: 16pt, Regular (main content)
Caption: 14pt, Regular (metadata, timestamps)
Button: 16pt, Semibold
Visual Design

Icons: Feather icons from @expo/vector-icons
Cards: White background, subtle 1px border, 12px border radius, no shadow
Role Badges: Pill-shaped, filled with role color (Staff: blue, Management: purple)
Floating Action Button: Primary color, 56x56, circular, shadow:
shadowOffset: {width: 0, height: 2}
shadowOpacity: 0.10
shadowRadius: 2
Input Fields: 1px border, 8px border radius, 12px padding, focus state shows primary color border
Destructive Actions: Nested in settings, red text, confirmation alerts required
Assets

Default Staff Avatars (6 options):
Professional geometric patterns in brand colors
Circular, 80x80px for profile, 40x40px for lists
Abstract professional designs (no emojis)
App Logo: Simple icon for header/splash
Empty States:
No messages illustration
No content illustration
No staff members illustration
Interaction Design

All touchable elements: 0.6 opacity when pressed
Form inputs: border color change on focus
List items: light gray background on press
PIN keypad: button scale down (0.95) on press
Error shake: horizontal translation animation for failed PIN
Success toast: slide down from top for confirmations
Pull-to-refresh: standard iOS/Android spinner
Accessibility

Minimum touch target: 44x44pt
Color contrast ratio: 4.5:1 minimum
Screen reader labels on all interactive elements
Dynamic type support for all text
Clear focus indicators for keyboard navigation
Role badges announced with semantic meaning
Destructive actions clearly labeled and confirmed
